# AI Agent 사전 과제 1-Page Summary (초안)

- **이름**: 최원교  **소속**: SK하이닉스
- **에이전트 이름**: 검증형 시험데이터 리포트 에이전트 (Report-Verify Agent)
- **에이전트 URL**: https://<배포후-앱이름>.streamlit.app  *(Streamlit Community Cloud 배포 후 기입)*
- **사용 Tool**: Python, LangGraph, Anthropic Claude API(claude-opus-4-8), Streamlit, Pydantic

## 01 에이전트 소개
시험·계측 데이터(CSV·현장 메모)를 업로드하면 **① 가드레일이 입력을 검증·차단 → ② 분석가가
측정값·기준·판정을 정리(없는 값은 '확인 필요') → ③ 작성자가 표준 양식으로 보고서 초안 작성 →
④ 독립 검증자가 원본과 대조해 PASS/FAIL 보고 → ⑤ 사람이 최종 승인**하는 멀티에이전트 파이프라인.
역할별 권한과 작업 수칙을 코드(하네스)로 고정해, 누가 실행해도 같은 품질의 보고서가 나오고
환각·출처 불명 수치를 자동으로 걸러낸다.

## 02 도입 효과
**AS-IS**
- 엔지니어가 측정 CSV를 눈으로 spec과 대조해 보고서를 수기 작성 (건당 수십 분)
- 작성자마다 표 양식·판정 기준 편차, 검토 누락으로 출처 없는 수치가 보고서에 섞임
- 작성자가 자기 보고서를 자기가 검토해 오류를 놓침(자기평가 편향)

**TO-BE**
- 업로드 한 번으로 분석·작성·검증까지 자동 처리 → 초안 작성 시간 **약 80%↓** (건당 수십 분 → 수 분)
- 작업 수칙을 코드로 고정 → 판정·양식 **편차 0**, "확인 필요" 규칙 자동 적용
- 독립 검증자가 출처 없는 파생값·수치 오류·양식 누락을 **자동 적발**(자기평가 편향 제거)
- 최종 승인은 사람이 수행 → 자동화와 책임의 분리 유지

## 03 스크린샷
- **메인 화면**: 시작 화면(파일 업로더 + '샘플 데이터로 실행' 버튼)
- **동작 화면**: 가드레일 통과 → 분석 표 → 검증자 PASS/FAIL → 보고서 초안 + 승인 버튼
